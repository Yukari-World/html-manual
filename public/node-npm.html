npm コマンド

説明

Node.jsのパッケージはコマンドプロンプトで操作を行う。その際によく利用されるコマンドを記述する。

コマンド

npm audit [--json]
npm audit fix [--force|--package-lock-only|--dry-run|--production|--only=(dev|prod)]

npm 6.0.0から登場したコマンド。
パッケージに存在する脆弱性を確認するコマンド。解決策がある場合は解決策が提示される。

npm dedupe
npm ddp

パッケージ毎に存在するパッケージの依存関係を整理する。パッケージ更新直後は大量のエラーが発生することが多いため、ここで整合性を整理する。

npm init [--force|-f|--yes|-y|--scope]
npm init &li;@scope> (same as `npx &li;@scope>/create`)
npm init [&li;@scope>/]<name> (same as `npx [&li;@scope>/]create-<name>`)

package.jsonの作成を行う。作成をする際、パッケージの名前、作成者、ライセンス等幾つか尋ねられる。

npm install <package> <option>
npm i <package> <option>

パッケージをインストールする。インストール済みパッケージで且バージョンが異なる場合は最新バージョンにアップデートする。
GithubのURL等を指定してnpmに登録されていないパッケージをインストールすることもできる。
npm installのみでインストールする場合、package.jsonからインストールする。

<option>

  • -g：グローバルパッケージとして保存する。
  • --save：パッケージとして保存する。
  • --save-dev：開発用パッケージとして保存する。
  • -D：開発用パッケージとして保存する。
npm list <option>
npm ls <option>

インストールされているパッケージとその依存関係を表示する。

npm outdated

パッケージリストにある更新のあるパッケージを表示する。更新は行われない。

npm prune

node_modulesに存在する不要なパッケージを削除する。

npm run-script <command>
npm run <command>

パッケージに記述されているスクリプトを起動する。ローカルに登録されているコマンドスクリプトは基本的にここから起動する。

npm substack

substackの名言を表示する。所謂イースターエッグの一つ。

npm update <package> <option>
npm upgrade <package> <option>
npm up <package> <option>

パッケージの更新を行う。

<option>

  • -g：グローバルパッケージに対して更新を行う。
  • -D：開発用パッケージに対して更新を行う。
npm uninstall <package>
npm un <package>

パッケージのアンインストールを行う。

npm visnup

(海外での)イジられキャラvisnupさんを生成する。所謂イースターエッグの一つ。

npm xmas

コマンドラインにクリスマスツリーを生成する。ツリーの飾りは生成毎に異なる。所謂イースターエッグの一つ。

便利なコマンド

覚えておくと便利なコマンドを記述する。

npm i <package>@<version> <option>

パッケージの特定のバージョンをインストールする。ベータ版をインストールする場合や最新版に特定の問題がある場合等に使用される。

npm list -g --depth=0

グローバルにインストールされている最上位階層のディレクトリを表示する。

npm update npm -g

npmコマンドをアップデートする。npm自体もパッケージの一つであり、時々更新されたりする。
