    EditorConfig

    EditorConfigとは

    EditorConfigとはエディター毎に存在するインデント設定や改行コードの設定等を統一する設定ファイルとそのプラグインである。また、ファイル毎に応じてその設定を変更することができ、また、プロジェクト毎に管理できるので記述ルールの整合性をある程度ではあるが保つことができる。

    導入方法

    エディターによってはデフォルトで対応している場合もあるが、プラグインを必要としているものもある。
    詳細な導入方法は別日記述予定。

    Atom

    コマンドラインからインストール

    コマンドラインに以下のコマンドを入力することでインストールできる。但し、Atomがマニュアルインストールの場合使用することができない。

    apm install editorconfig

    パッケージからインストール

    EditorConfigと検索すると結果に出てくるのでそれをインストールする。

    インストールにおける注意点

    whitespace(Atomのデフォルトパッケージ)のパッケージが競合するため、設定変更、もしくは無効化をすること。

    インデントの設定をautoに設定すること。

    設定ファイル

    設定ファイルは.editorconfigファイルに記述する。

    .editorconfigの例

    # EditorConfigの最上位ファイルであることを示す
root = true

# 全体共通設定
[*]
charset = utf-8
end_of_line = lf
indent_size = 4
indent_style = tab
insert_final_newline = true
trim_trailing_whitespace = true

# PHPファイル全体に適応する内容
[*.php]
indent_style = space

# ファイル名を直接指定することもできる
[package.json]
indent_size = 2

    設定は上から下へと適応される。

    設定項目の説明

    indent_style

    TABキーを入力した時のインデント方法を指定する。指定可能な項目は以下の通り。

    • tab
    • space

    indent_size

    インデントサイズを指定する。

    tab_width

    タブのインデントサイズを指定する。未指定の場合、indent_sizeの値が適応される。

    end_of_line

    ファイルの改行コードを指定する。指定可能な項目は以下の通り。

    • lf
    • cr
    • crlf

    charset

    ファイルの文字コードを指定する。指定可能な項目は以下の通り。

    • latin1
    • utf-8
    • utf-8-bom
    • utf-16be
    • utf-16le

    trim_trailing_whitespace

    行末にあるスペースやタブといった空白を削除するかどうか。trueに指定すると行末の空白を削る。指定可能な項目は以下の通り。

    • true
    • false

    insert_final_newline

    最終行に空白の行を追加するかどうか。指定可能な項目は以下の通り。

    • true
    • false

    root

    このコンフィグファイルが最上階であるかどうか。falseの場合、ルートディレクトリまで遡って.editorconfigファイルを捜索する。指定可能な項目は以下の通り。

    • true
    • false
