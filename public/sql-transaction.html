      TRANSACTION(トランザクション)

      説明

      INSERT, UPDATE, DELETEといったSQLコマンドはデータベースの変動を伴うため、正確に記述しなければならない。しかし、テスト環境を構築した上で、データを崩さずに確認したい場合がある。そのときに使えるコマンドである。
      また、更新などで複数のデータテーブルの内容を更新を行う時、更新処理を行う途中でエラー等で処理を中断する際に発生するデータテーブルの一部は更新したが一部は更新しなかったことで起こるデータの不整合を防ぐ事にも使用できる。

      使用方法と解説

      COMMIT

      COMMITとは処理を確定させるために使う。使用方法は至って単純である。

      START TRANSACTION;
-- ~~~ SQL CODE ~~~
COMMIT;

      トランザクションを挟むことで複数の処理で一つの処理として取り扱われる。

      ROLLBACK

      ROLLBACKとは、データベースを更新前のデータに戻す時に使用される。こちらも使用方法は至って単純である。

      START TRANSACTION;
-- ~~~ SQL CODE ~~~
ROLLBACK;

      尚、トランザクションを省略した時の通常動作はCOMMITで動作する。

      使用上の注意

      • ロールバックされるまでSQL実行結果は保持される。INSERTやUPDATE、DELETEを行った後に SELECT文を実行することで反映されたデータテーブルを確認できる。
      • ロールバックされるからと言って間違っても本番環境で使用しないように注意。START TRANSACTION;を入力することを忘れた場合、効果がないためである。
      • AUTO_INCREMENTはSQLの結果がロールバックされる場合でも通常通りカウントされ、元に戻ることはない。このためID番号にて不揃いが発生する可能性がある。
      • 一部のSQLコマンドはトランザクションの影響を受けない。これは実行後自動的にCOMMIT動作が実行されるためである。
      • PHPでSQLトランザクションを行う場合、PHP側にその処理に該当する関数が存在するのでそちらを使用すると良い。
